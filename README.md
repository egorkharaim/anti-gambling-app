# Bankroll Discipline Manager

## Описание проекта

Bankroll Discipline Manager — персональное web-приложение для строгого учета
игрового бюджета. Оно не блокирует сайты и не принимает решения за пользователя,
но заранее фиксирует лимиты, показывает их фактическое использование и сообщает о
нарушениях без осуждающих формулировок.

Главная цель первой версии — корректный финансовый учет. Авторизация, облачная
синхронизация, платежи, социальные функции и интеграции с казино в MVP не входят.

Полный порядок разработки, модели, маршруты и критерии готовности описаны в
[DEVELOPMENT_PLAN.md](DEVELOPMENT_PLAN.md).

## Основные правила продукта

- Денежные суммы представлены типом `Money`, внутри которого используются
  `BigDecimal` и код валюты.
- Для каждой валюты фиксируется допустимый scale; неявное округление запрещено.
- Создавать денежный `BigDecimal` из `double` запрещено.
- Пользователь выбирает валюту профиля.
- После появления первого плана валюту нельзя изменить без создания нового профиля.
- Активный недельный или месячный лимит можно только уменьшить.
- Уменьшение лимита не изменяет прошлые операции и результаты.
- Выигрыш не восстанавливает лимит потерь текущего периода.
- Новую сумму пользователь определяет только для следующего периода.
- Превышения разрешено записывать: приложение предупреждает, но не блокирует учет.

## Что такое bankroll-domain

`bankroll-domain` — независимое расчётное ядро приложения. В нём находятся
`Money`, финансовый профиль, планы, лимиты, игровые сессии и правила
предупреждений. Этот модуль ничего не знает о Spring, HTTP, JavaScript и базе
данных. Благодаря этому финансовые правила можно тестировать отдельно, а
web-интерфейс не сможет случайно подменить их своими расчётами.

## Командная работа

1. `main` всегда должна собираться и проходить тесты.
2. Каждая задача выполняется в отдельной feature-ветке.
3. Изменения попадают в `main` только через Pull Request.
4. Второй разработчик проверяет PR до слияния.
5. Один PR решает одну законченную задачу и не содержит постороннего рефакторинга.
6. Изменения финансовых формул обязательно сопровождаются тестами.

Рекомендуемый формат веток:

- `infra/project-setup`
- `domain/money`
- `domain/bankroll-plan`
- `api/sessions`
- `ui/dashboard`
- `fix/limit-calculation`

Рекомендуемый формат коммитов:

- `Add currency-safe Money value object`
- `Implement active plan decrease rule`
- `Fix loss budget calculation for winning sessions`
